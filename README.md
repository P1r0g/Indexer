# 📄 Text File Indexer

Библиотека для индексации текстовых файлов по словам с поддержкой поиска и отслеживания изменений файловой системы.

---

## 🚀 Возможности

* 📁 Индексация отдельных файлов и директорий (рекурсивно)
* 🔍 Поиск файлов, содержащих заданное слово
* 🔄 Автоматическое обновление индекса при:

  * создании файлов
  * изменении файлов
  * удалении файлов
  * изменении структуры каталогов
* ⚙️ Расширяемый механизм разбиения текста на слова (Tokenizer)
* 🧵 Потокобезопасный доступ к индексу
* 💻 Консольное приложение (CLI) для работы с библиотекой

---

## 🏗 Архитектура

Основные компоненты:

* **FileIndexService** — публичный API библиотеки
* **InMemoryFileIndexService** — реализация сервиса индексации
* **IndexStorage** — хранение индекса в памяти
* **Tokenizer** — интерфейс разбиения текста на слова
* **SimpleTextTokenizer** — простая реализация токенизации
* **FileSystemWatcher** — отслеживание изменений файловой системы

### Структура индекса

Индекс хранится в двух структурах:

* `word -> files` (слово → список файлов)
* `file -> words` (файл → список слов)

Это позволяет:

* быстро находить файлы по слову
* корректно обновлять индекс при изменении файлов

## 💻 Использование CLI

После запуска доступны команды:

```text
add-file <path>   - добавить файл в индекс
add-dir <path>    - добавить директорию (рекурсивно)
search <word>     - найти файлы по слову
help              - показать список команд
exit              - выйти из программы
```

### Пример использования:

```text
> add-dir sample-data
Directory indexed and watcher registered.

> search java
Found in:
 - /path/to/a.txt
 - /path/to/b.txt
```

---

## 🔍 Принцип работы

1. Файл читается как текст (UTF-8)
2. Текст разбивается на слова с помощью `Tokenizer`
3. Все слова приводятся к нижнему регистру
4. Формируется индекс:

```text
java -> [a.txt, b.txt]
maven -> [a.txt]
```

---

## 🔄 Отслеживание изменений

После добавления директории:

* запускается `WatchService`
* отслеживаются события:

  * создание файла
  * изменение файла
  * удаление файла

При изменениях индекс автоматически обновляется без перезапуска программы.

---

## 🧵 Потокобезопасность

Для обеспечения корректной работы в многопоточном окружении используется `ReadWriteLock`:

* 🔓 `readLock` — для операций поиска
* 🔒 `writeLock` — для изменения индекса

Это позволяет:

* выполнять несколько поисков параллельно
* гарантировать консистентность данных

---

## 🔧 Расширяемость

Механизм токенизации вынесен в отдельный интерфейс:

```java
public interface Tokenizer {
    Set<String> tokenize(String text);
}
```

Это позволяет легко добавлять новые реализации:

* RegexTokenizer
* Lexer-based tokenizer
* специализированные парсеры (например, для исходного кода)

---

## ⚠️ Ограничения

* Индекс хранится только в памяти (не сохраняется между запусками)
* Поиск выполняется по точному совпадению слова
* Не реализованы stemming и лемматизация
* Все файлы читаются как UTF-8
* Возможны дублирующиеся события от файловой системы (обрабатываются безопасно)
